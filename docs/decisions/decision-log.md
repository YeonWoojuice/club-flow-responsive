# Decision Log

> 프로젝트의 굵직한 결정과 그 이유를 기록한다.
> 나중에 "왜 이렇게 했더라?"가 나오면 이 파일부터 본다.
> 형식: 결정 / 이유 / 버린 대안. 결정을 뒤집으면 지우지 말고 새 항목으로 추가한다.

## 2026-07-08 ~ 07-09 · 협업 구조 개편

### 1. 분업 기준: 도구별 레이어 → 기능(수직) 단위

- **결정**: "UI는 Claude, 백엔드는 Codex" 방식을 버리고, 한 기능을 한 도구가 프론트+백 끝까지 맡는다.
- **이유**: 경계선이 코드 한가운데(API 계약)를 지나가면 양쪽이 각자의 방언으로 진화한다.
  실제로 에러 처리 분열, `Input` vs `Request` 네이밍, phone 필수 여부 불일치가 발견됐다.
- **버린 대안**: 레이어 분업 유지 + 자동화 보완 — 타입은 지켜주지만 스타일·철학 분열은 못 막는다.

### 2. AI 역할: Claude = 설계·리뷰·막힌 문제, Codex = 구현

- **결정**: Claude가 설계를 문서로 남기면 Codex가 그 문서와 AGENTS.md만 읽고 구현한다.
  계약 변경·인증·마이그레이션은 완료 후 Claude 리뷰를 거친다.
- **이유**: 1. Claude 토큰 소모가 빠르다 → 비싼 도구는 판단이 필요한 일에만.
  2. 두 도구를 대화로 중계시키는 노동이 컸다 → 전달을 대화가 아닌 문서로 바꿔 중계 비용 제거.
- **버린 대안**:
  - 기능별 전담 + 교차 리뷰 — 핸드오프마다 사용자가 중계해야 해서 기각.
  - 단일 도구 통일 — 두 번째 시각을 잃고, 비용 분산도 안 됨.
  - Claude 단독 — Cowork 샌드박스의 실행 한계(테스트·빌드를 사용자에게 떠넘김)와 토큰 비용.

### 3. 규칙 문서: AGENTS.md 단일 소스

- **결정**: AGENTS.md가 모든 규칙·구조·용어의 원본. CLAUDE.md는 참조 스텁으로 축소.
  기술 스택 버전은 AGENTS.md 표에만 적고 나머지 문서는 링크.
- **이유**: 어떤 도구가 와도 같은 문서를 읽으면 규칙이 도구별로 갈라질 통로가 닫힌다.
  실제로 스택 표가 4개 문서에 중복돼 "Spring Boot 4" vs "4.0.7"로 이미 어긋나 있었다.
- **버린 대안**: 도구별 지침 파일 유지 — frontend-rules가 "claude sonnet용"으로 진화한 것이 분열의 증거.

### 4. API 계약: OpenAPI가 원본, 프론트 타입은 생성 (`npm run gen:api`)

- **결정**: 백엔드 `/v3/api-docs`에서 TS 타입을 기계적으로 생성한다. 손으로 미러링하지 않는다.
- **이유**: 같은 계약이 두 파일에 손으로 쓰여 있으면 누가 작업하든 한쪽을 깜빡할 수 있다.
  단일 도구 구현으로 바꾼 뒤에도 유지 — 이 장치가 막는 건 도구 간이 아니라 **파일 간** 어긋남이다.
- **버린 대안**: 수동 타입 유지 + 조심 — phone 불일치가 이미 그 방식의 실패 사례.

### 5. 에러 문구의 단일 소스 = 백엔드

- **결정**: 백엔드가 사용자용 한국어 `message`를 만들고, 프론트는 조회·쓰기 모두 그걸 우선 표시한다.
  프론트 분기는 HTTP status로만, `code`는 로깅용.
- **이유**: 에러 문구를 양쪽에서 관리하면 중복. 백엔드는 이미 잘 만들고 있었는데 프론트 조회 경로가 버리고 있었다.

### 6. 네이밍은 백엔드 도메인 명칭 기준

- **결정**: `ClubRole` → `ClubStaffRole` 등 데이터 모델 문서와 백엔드 명칭으로 통일.
- **이유**: 도메인의 원본이 데이터 모델(`club_staffs.role`)이므로.

### 7. docs 구조: 기능 관통 문서는 `docs/features/`

- **결정**: `docs/auth` → `docs/features/auth` 이동. 앞으로 기능 흐름 문서는 여기에.
- **이유**: 기존 docs는 성격별(product/development)인데 auth만 기능별이라 기준이 섞여 있었다.

### 8. FE `features/` 폴더 재구성은 보류

- **결정**: `src/api·types·pages` 레이어 구조를 당장 바꾸지 않는다.
- **이유**: 파일 40여 개 규모에선 이동 비용 대비 실익이 작다. gen:api가 타입 동기화 고통을 이미 제거.

### 9. 완료 정의 (Definition of Done)

- **결정**: `./verify.sh` 통과 없이 완료 선언 금지. 실행 불가 환경이면 "미검증" 라벨 + 사용자가 돌릴 명령 명시.
- **이유**: AI가 검증을 조용히 사용자에게 떠넘기는 경향을 규칙 위반으로 만들기 위해.
  (2026-07-08 Cowork 세션에서 Claude가 테스트를 사용자에게 넘긴 것이 계기.)

### 10. 미결 사항

- 디자인 기준 파일을 `club_flow_v2.pen`으로 표기한 것은 파일 날짜 기준 **추정** — 사용자 확인 필요.
- 루트의 `club_project` 파일 정체 확인 필요.
