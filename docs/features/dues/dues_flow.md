# 회비 관리 흐름

## 금액 규칙

- 한 학기에는 회원·준회원·비활동 부원에게 같은 회비를 부과한다.
- 금액은 DB `NUMERIC(19,0)`, 백엔드 `BigDecimal`, API 문자열로 처리한다.
- 학기 합계는 저장하지 않고 유효한 납부·환불 기록을 합산한다.
- 환불 계산에서 원 미만 금액은 버린다(`RoundingMode.FLOOR`).
- 탈퇴일은 클라이언트 입력을 받지 않고 부원 상태 변경 이력의 `WITHDRAWN` 시각을 한국 시간 날짜로 변환한다.

## 처리 흐름

```text
학기 회비와 환불 기준 설정
  → 현재 부원별 부과 기록 생성
  → 기존 PAID/EXEMPT 상태 이관
  → 납부 확인 또는 면제 처리
  → 탈퇴 부원의 서버 탈퇴일로 환불액 계산
  → 실제 송금 후 환불 완료 기록
```

- 환불 기준은 `기준일(포함) + 환불 비율` 목록이며 마지막 기준일 이후는 0%다.
- 납부와 환불 요청은 고유 요청 키로 중복 저장을 막는다.
- 같은 부원에게 유효한 납부와 환불은 각각 한 건만 존재한다.
- 회비 설정 후 새로 생성되거나 이월된 부원도 자동으로 부과 대상에 포함한다.
- 회비 정책이 있는 학기는 부원 목록의 단순 회비 상태가 아니라 회비 관리 화면에서 처리한다.

## 기존 상태 이관

- `PAID`: 설정된 학기 회비 전액의 기존 확인 납부 기록 생성
- `EXEMPT`: 면제 사유를 `기존 면제 상태 이관`으로 기록
- `UNKNOWN`, `UNPAID`: 미납 부과 기록 생성

기존 납부 기록에는 실제 납부일 정보가 없으므로 `paid_on`은 비워 두고 `LEGACY_STATUS` 출처로 구분한다.
